Additional static configuration can be loaded from YAML files.
The files have the same structure as a Kubernetes manifest.

Usage

  yat serve -config PATH [-config PATH]

RuleSet Example (unimplemented)

  apiVersion: yat.io/v1alpha1
  kind: RuleSet

  metadata:
    name: example

  rules:
    # allows everything
    # if you can connect,
    # you can pubsub
    - allow:
        - path: "**"
          actions: [pub, sub]
